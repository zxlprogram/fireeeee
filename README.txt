# Fireeeee — 火災求生模擬器（有系統 vs 無系統）

簡短說明
- Fireeeee 是一套以 Java 實作的火災逃生模擬器，用來評估「沒有偵測/資訊系統（無系統）」與「部署偵測器並分享資訊的智慧支援系統（有系統）」在同一場景下對受難者存活率與存活時間的差異。
- 本專案以模擬結果量化系統效益，並提供可重複、可參數化的大量蒙地卡羅模擬平台（用以驗證系統設計假說）。

重點結論（數值總結）
- 無系統情況的平均逃生率：約 1.8%。
- 有系統情況的平均逃生率：約 15.6%。
- 系統對「每位個體平均存活時長」的優化程度與火勢蔓延速率成正比，實驗得到的比例常數約為 5.734（即在本模型與參數下，火勢速率增加時，系統帶來的時間優化大致以此常數尺度變化）。

模擬方式（方法說明）
- 比較目標：在同樣「沒有消防車、沒有滅火器、火勢按固定規則蔓延」的情況下，比較有無系統下「存活率（生還人數/總人數）」與「存活時間」。
- 關鍵觀察變數：本模擬把系統效益視為「使用者會知道哪些資訊（感測器回報/全域共享）」，並依此影響每一步中個體的決策（局部視野 vs 全域訊息）。
- 每次模擬流程：
  1. 隨機產生建築（樓層、牆、門、樓梯、單一出口）。
  2. 放置火源（單一格）並啟動火/煙擴散規則。
  3. 放置受難者（數與速度分佈可參數化）。
  4. 兩組實驗：一組為「無系統」行為（局部視野、遇濃煙視野受限、遇煙採隨機移動等）；另一組為「有系統」行為（偵測器佈署、偵測器資訊全域可得、使用基於偵測器與可見格整合的路徑規劃與避險策略）。
  5. 以 tick 為單位運行至建築「被火焚毀（所有非牆格著火）」或所有人皆逃生/死亡為止。

模擬假設與缺陷（重要限制）
- 無外部滅火資源：模擬中不含消防車、滅火器、外部救援的滅火行為。
- 火勢行為：火以固定規則在每個 tick 向四方向擴散（每隔 tick 擴散）；不模擬風或複雜熱流。
- 煙霧行為：煙在每 tick 從來源向距離1處 +0.2、距離2處 +0.1，來源格也累加；相加疊加；可設定最大值 1.0。
- 偵測器規則：平均每約 3 平方格放一個 detector；若偵測器所在格著火，該偵測器失效（壞掉）。
- 行為模型簡化：人類行為以啟發式（局部 BFS、隨機逃逸、或基於偵測器的 A*/BFS）實作；心理/群體行為僅用簡化的情緒/隨機規則模擬。
- 以上假設會影響絕對數值，但對「有系統 vs 無系統」的相對差異仍具指標性。

量化方式（指標定義）
- 逃生率（Survival rate）= 生還人數 / 總人數。
- 個體存活時長定義：
  - 若個體逃出：存活時長 = 房屋從起火到「整棟房屋被完全燒毀」的總時長（也就是模擬結束時的 tick）。
  - 若個體被火燒死：存活時長 = 從起火到該個體死亡的 tick。
- 系統優化值（本次報告呈現）：
  - 對每個人計算 (存活時長_with_system − 存活時長_without_system)，求全體個體的平均值。報告中指出此平均值隨火勢蔓延速率變化成正比，比例常數為 5.734（基於本模擬模型與參數得到）。

量化結果（本次模擬摘要）
- 在所選的測試集（多次隨機場景）與模型參數下：
  - 無系統的整體平均逃生率：約 1.8%（代表沒有偵測器/資訊支援時的極低逃生機率）。
  - 有系統的整體平均逃生率：約 15.6%（部署偵測器並共享資訊後，逃生率顯著上升）。
  - 系統對「每位個體平均存活時長」的優化量不是線性增長，而傾向於一個與火勢速率成比例的常數尺度（本實驗得 5.734）。

如何執行模擬（快速開始）
1. 編譯
   - 以專案根目錄為工作目錄：
     javac -d out src/simulator/*.java
2. 執行
   - 執行主程式（在 sim.main 可調參數）：
     java -cp out simulator.sim
   - 在 main 中，可透過呼叫 s.work(iterations, new Range(Hmin,Hmax), new Range(Rmin,Rmax), new Range(Cmin,Cmax)) 調整模擬次數與地圖尺寸範圍。
   - 範例（程式原預設）：
     s.work(100, new Range(1,11), new Range(5,50), new Range(5,50));
3. 輸出檔案
   - 每次 run 會產生兩個過程檔案：`defaultResult.txt`（無系統過程）與 `smartResult.txt`（有系統過程）。僅會保留最後一次模擬的過程檔案（每次模擬會被覆蓋），若需保留所有模擬，請在呼叫 run 前修改輸出檔名或加入 timestamp。
   - 統計結果（平均逃生率、優化程度等）會輸出到標準錯誤（stderr），便於在自動化批次實驗中蒐集。

建議輸出格式（便於後續分析）
- results/summary_<timestamp>.csv — 每次模擬總表（場景參數、總人數、生還人數、死亡人數、平均生存時長、逃生率）。
- results/agents_<timestamp>.csv — 每位 agent 的紀錄（起始座標、速度、逃生/死亡 tick、逃生狀態）。
- results/map_tick_<simid>_<tick>.txt — 每 tick 的地圖快照（文字化）。
- results/full_<simid>.json — 完整事件紀錄（若需做視覺化或更多統計分析，建議新增）。

解讀建議與局限
- 本模擬提供「比較性」而非「絕對值」的證據：絕對逃生率會依建築配置、人口密度、火源位置、門/牆分佈與偵測器密度顯著改變；但在相同假設條件下數千次蒙地卡羅實驗所得的相對改善（從 ~1.8% 提升到 ~15.6%）顯示資訊共享與智慧決策支援可以顯著降低死亡風險。
- 模型簡化了真實世界物理（風、溫度、一氧化碳致死等）與群體心理，未包含主動救援隊伍與外部滅火，這些會影響實際部署的數值結果。
- 「比例常數 5.734」是本模型在所用參數與火勢規則下的實驗性結果，並非普適常數；若改變火勢擴散速率或偵測器密度，此常數會變動。建議做敏感度分析以取得更穩健的估計範圍。
