## 簡短說明

- Fireeeee 是一套以 Java 實作的火災逃生模擬器，用來評估「沒有偵測/資訊系統（無系統）」與「部署偵測器並分享資訊的智慧支援系統（有系統）」在火災環境下的逃生效率差異。
- 本專案以模擬結果量化系統效益，並提供可重複、可參數化的大量蒙地卡羅模擬平台（用以驗證系統設計假說）。

## 重點結論（數值總結）

- 無系統情況的平均逃生率：約  3.33%。
- 有系統情況的平均逃生率：約 62.96%。
- 系統對「每位個體平均存活時長」的優化程度與火勢蔓延速率成正比，實驗得到的比例常數約為 31.705（即在本模型與參數下，火勢速率增加時，系統的優化效益也隨之提升）。

## 模擬方式（方法說明）

- **比較目標**: 在同樣「沒有消防車、沒有滅火器、火勢按固定規則蔓延」的情況下，比較有無系統下「存活率（生還人數/總人數）」與「存活時間」的差異。
- **關鍵觀察變數**: 本模擬把系統效益視為「使用者會知道哪些資訊（感測器回報/全域共享）」，並依此影響每一步中個體的決策（局部視野 vs 全域訊息）。
- **每次模擬流程**:
  1. 隨機產生建築（樓層、牆、門、樓梯、單一出口）。
  2. 放置火源（單一格）並啟動火/煙擴散規則。
  3. 放置受難者（數與速度分佈可參數化）。
  4. 兩組實驗：一組為「無系統」行為（局部視野、遇濃煙視野受限、遇煙採隨機移動等）；另一組為「有系統」行為（偵測器佈署、偵測器資訊全體共享）。
  5. 以 tick 為單位運行至建築「被火焚毀（所有非牆格著火）」或所有人皆逃生/死亡為止。

## 模擬假設與缺陷（重要限制）

- **無外部滅火資源**: 模擬中不含消防車、滅火器、外部救援的滅火行為。
- **火勢行為**: 火以固定規則在每個 tick 向四方向擴散（每隔 tick 擴散）；不模擬風或複雜熱流。
- **煙霧行為**: 見最新更新內容。
- **偵測器規則**: 平均每約 3 平方格放一個 detector；若偵測器所在格著火，該偵測器失效（壞掉），若偵測器所在格檢測濃煙濃度大於0.7則觸發危險警示。
- **行為模型簡化**: 人類行為以啟發式（局部 BFS、隨機逃逸、或基於偵測器的 A*/BFS）實作；心理/群體行為僅用簡化的情緒/隨機規則模擬。
- **以上假設會影響絕對數值，但對「有系統 vs 無系統」的相對差異仍具指標性。**

## 量化方式（指標定義）

- **逃生率（Survival rate）** = 生還人數 / 總人數。
- **個體存活時長定義**:
  - 若個體逃出：存活時長 = 房屋從起火到「整棟房屋被完全燒毀」的總時長（也就是模擬結束時的 tick）。
  - 若個體被火燒死：存活時長 = 從起火到該個體死亡的 tick。
- **系統優化值（本次報告呈現）**:
  - 經觀察，在不同的無系統生還率下，有系統的生還率與無系統生還率普遍趨近於 `y=f(m)(x+k)` 的常數差距關係，其中，`k`為常數，代表有系統對初始無系統情況改善的量，`f(m)`則是不同樓型對生還率的影響常數，`f(m)`並非單一參數的函數，而是簡化多個影響因素(例如人數、樓層數、樓長與寬、火源位置、門口位置等等)後的表示。
  - 對每個人計算 (存活時長_with_system − 存活時長_without_system)，求全體個體的平均值。報告中指出此平均值隨火勢蔓延速率變化成正比，比例常數為 31.705。

## 量化結果（本次模擬摘要）

在所選的測試集（多次隨機場景，從小套房到十層百貨公司大樓）與模型參數下：

- 無系統的整體平均逃生率：約 3.33%（代表沒有偵測器/資訊支援時的極低逃生機率）。
- 有系統的整體平均逃生率：約 62.96%（部署偵測器並共享資訊後，逃生率顯著上升）。
- 系統對「每位個體平均存活時長」的優化量不是線性增長，而傾向於一個與火勢速率成比例的常數尺度（本實驗得 31.705）。
- 有系統在小樓型中的準確率為97%至98.5%，優化常數約為9.44，在大樓型中，因為運行時間過長，僅透過百次隨機模擬與小樓型數據，估計準確率較低，優化常數則較高（在單筆大樓型的測試資料中，優化常數達96.471，細節見報告文檔），代表在大樓型中，系統相對於小樓型，更傾向讓人積極地躲避火源。

## 如何執行模擬（快速開始）

### 1. 編譯

以專案根目錄為工作目錄：

```bash
javac -d out src/simulator/*.java
```

### 2. 執行

執行主程式（在 sim.main 可調參數）：

```bash
java -cp out simulator.sim
```

在 main 中，可透過呼叫 `s.work(iterations, new Range(Hmin,Hmax), new Range(Rmin,Rmax), new Range(Cmin,Cmax))` 調整模擬次數與地圖尺寸範圍。

範例（程式原預設）：

```java
s.work(100, new Range(1,11), new Range(5,50), new Range(5,50));
```

### 3. 輸出檔案

- 每次 run 會產生兩個過程檔案：`defaultResult.txt`（無系統過程）與 `smartResult.txt`（有系統過程）。僅會保留最後一次模擬的過程檔案。
- 統計結果（平均逃生率、優化程度等）會輸出到標準錯誤（stderr），便於在自動化批次實驗中蒐集。

## 解讀建議與局限

- 本模擬提供「比較性」而非「絕對值」的證據：絕對逃生率會依建築配置、人口密度、火源位置、門/牆分佈與偵測器密度顯著改變；但在相同假設下的「有系統 vs 無系統」相對差異仍具參考價值。
- 模型簡化了真實世界物理（風、溫度、一氧化碳致死等）與群體心理，未包含主動救援隊伍與外部滅火，這些會影響實際部署的數值結果。
- 「比例常數 31.705」是本模型在所用參數與火勢規則下的實驗性結果，並非普適常數；若改變火勢擴散速率或偵測器密度，此常數會變動。建議做敏感度分析。
